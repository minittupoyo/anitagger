# 🏷️ anitagger (アニタガー)

anitagger は、国産アニメデータベースサービス **Annict (アニクト)** の豊富なデータを活用し、ローカルのアニメ動画ファイルを自動的かつ直感的にリネームする、Tauri & Vue 3 製のモダンなデスクトップアプリケーションです。

---

## ✨ 主な機能

### 1. 🔍 Annict API 連携による作品・エピソード検索
* 左サイドバーからアニメタイトルを入力するだけで、Annict に登録されている作品情報（タイトル、メディア種別、放送シーズン）を瞬時に検索します。
* 作品選択後、全エピソード情報（話数、サブタイトル等）を自動取得します（50話を超える長編作品のページネーション取得にも対応）。

### 2. ⚡ 高度な話数自動マッチング
* ファイル名に施された様々な表記パターンから、正規表現を用いてエピソード番号を自動抽出します。
  * **対応パターン例**:
    * `S01E03` / `E03` / `E 03`
    * `ep03` / `episode 3`
    * `第03話` / `第 03 話`
    * `#03` / `- 03`
    * `[03]` / `(03)` など
* 抽出した話数と Annict のエピソード情報を照合し、`{話数} - {サブタイトル}.{拡張子}`（例: `01 - 始まりの物語.mp4`）のような最適なファイル名を自動で提案します。

### 3. 🎬 劇場版（映画）＆ OVA 作品への対応
* TVアニメだけでなく、エピソードリストが存在しない（または1話のみの）劇場版や OVA にも対応しています。
* 単一ファイル時は `作品名 - サブタイトル.ext`、複数ファイル時は `作品名 - 連番.ext` のように状況に合わせた柔軟なリネーム処理を行います。

### 4. 🔀 ドラッグ＆ドロップによる並び替え（ファイルマッピング）
* ファイル名とエピソードの自動マッチングがズレている場合や、ファイル名に話数が含まれていない場合でも安心です。
* リスト上のファイルをドラッグ＆ドロップ（または上下の矢印ボタン）で並び替えることで、変更前のファイルを希望するエピソード順に手動でマッピングし直すことができます。

### 5. ✏️ 変更後ファイル名の手動インライン編集
* 各タスクの「変更後」のタイトル部分にある鉛筆マーク ✏️ をクリックすることで、プレビュー上で個別にファイル名を手動編集できます。
* OSで禁止されているファイル名文字（`\`, `/`, `:`, `*`, `?`, `"`, `<`, `>`, `|`）は自動で `_` にサニタイズされます。

### 6. 🛡️ 安全な一括プレビューと選択実行
* 実際にファイルを書き換える前に、すべての変更内容をグリッドで比較確認できます。
* チェックボックス（「すべて選択」「選択解除」対応）により、特定ファイルのみをリネーム対象から除外することが可能です。
* 変更が発生するファイルには「変更あり」バッジが表示され、誤操作を防ぎます。

## 🛠️ 開発・ビルド方法

このプロジェクトは、フロントエンドに **Vue 3 (Vite + daisyUI + Tailwind CSS)**、バックエンドに **Tauri v2 (Rust)**、パッケージマネージャに **Bun** を使用して構築されています。

### 依存関係のインストール

```bash
bun install
```

### 開発モードの起動

```bash
bun tauri dev
```

### プロダクションビルド

```bash
bun tauri build
```
> [!IMPORTANT]
> - ビルドには Rust のコンパイル環境（Rustup 等）が必要です。
> - ビルドおよび実行には Annict の API トークンが必要です。`.env` または環境変数 `ANNICT_TOKEN` を設定した状態でビルドを実行してください。

---

## 📝 必要環境

* **OS**: Windows / macOS / Linux (Tauri がサポートするプラットフォーム)
* **APIキー**: アプリを使用する前に [Annict](https://annict.com/) にて個人用アクセストークン（読み込み権限）を発行し、ビルド時に注入する必要があります。
